# Delivery Plan

## Phase 1: Document the repo

- add the project, agent, architecture, design, and plan docs
- keep the docs grounded in the current app rather than future fantasy state
- make the visual direction explicit so future changes stay consistent

## Phase 2: Normalize the visual system

- shift shared theme tokens toward a mint/green palette
- align the landing page and core app shell with the new direction
- preserve contrast and readability across dark surfaces

## Phase 3: Validate the app

- check the affected screens for visual regressions
- keep demo mode and Spotify flows intact
- verify the updated palette still reads as calm, technical, and polished

## Phase 4: Iterate

- tighten any inconsistent green accents
- refine shared components if they still reference the old cyan language
- extend the same visual rules to additional screens as needed
